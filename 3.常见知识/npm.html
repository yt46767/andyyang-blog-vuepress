# 已有npm包，指定版本更新 [注意点]

  • npm插件如果原先安装好的，直接npm i 插件@版本。不用指向devDependences/dependences。

  • npm插件指定版本安装后，package.json会把插件更改为<i style="color:green">自动更新模式</i>

&quot;@company/fe-git-cli&quot;: &quot;1.0.54&quot;,

更改为：

&quot;@github/fe-git-cli&quot;: &quot;^1.0.58&quot;,

# 安装本地npm包

npm install --save ../package

# package-lock.json如果不删除npm包，会影响运行，去安装删除的npm包后再运行

# npm包引用策略：

引用的npm包，优先考虑全局的npm包，全局找不到才会去找项目本地的npm包

# 安装npm包，默认是下次安装，自动升级

wnpm i @company/fe-cli -D "@company/fe-cli": "^1.8.3",

# 查看npm全局-g安装过哪些包

npm ls -g -depth=0

# package.json引用git地址

"dependencies": { "public": "git://github.com/user/repo.git#ref", "private": "git+ssh://git@github.com:user/repo.git#ref" } 注： git+ssh://git@github.com:user/repo.git#commitid

# npm install git 仓库 的 某个commit id

wnpm i @github/power-cli@git+ssh://git@github.com:cfpd/power-cli.git#638a0edccaae3c00d6d9909fdd887482965dbb68 -g

# npm代理设置

Node开发环境现在已经可以直接通过npm进行安装依赖包了，但是需要重新设置依赖仓库，流程如下，请需要的人自行配置。 平安科技依赖仓库地址：[代理地址] Step1: 首先需要本地开发环境安装nodejs，这个请各个小组自行申请安装，建议安装最新的稳定版本。 Step2: 重置npm依赖库配置 打开命令行，执行 npm config set registry [代理地址] 执行npm config get registry查看执行结果 使用npm正常安装相关插件即可
